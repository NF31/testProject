## 🟦 Phase 1 – Conception & Planification

**Objectif :** Poser les bases du projet

- [ ] Définir le concept du jeu (Jump & Run, thème Toulouse)
- [ ] Lister les monuments à intégrer
- [ ] Dessiner le layout sur papier *(obligatoire selon le PDF)*
- [ ] Définir l’interface et la structure des écrans
- [ ] Réfléchir à la logique : saut, score, collisions, monuments, effet de course
- [ ] Répartition des tâches (Jean, Zindar, Noah)
- [ ] Créer la structure des dossiers du projet
- [ ] Regarder des tutos (Flappy Bird, Google Dinosaur)
- [ ] Explorer des repos GitHub pour inspiration

---

## 🟩 Phase 2 – Design & Assets

**Objectif :** Préparer tous les visuels et sons

- [ ] Créer le sprite de la chocolatine (classique, blanc, dorée)
- [ ] Dessiner les obstacles (monuments, ballon rugby, Beluga)
- [ ] Créer les décors de fond par niveau
- [ ] Trouver ou créer les images des monuments toulousains
- [ ] Choisir/créer les effets sonores (saut, collision, victoire)
- [ ] Choisir la musique de fond
- [ ] Définir la palette de couleurs et le style visuel
- [ ] Exporter tous les assets dans `assets/images/` et `assets/sounds/`

---

## 🟨 Phase 3 – Développement Core

**Objectif :** Faire tourner le jeu de base

- [ ] Créer `index.html` (écran de démarrage)
- [ ] Créer `game.html` (page du jeu)
- [ ] Mettre en place le CSS de base (`style.css`, `game.css`)
- [ ] Implémenter la boucle de jeu principale (`main.js`)
- [ ] Coder la physique du saut (simple)
- [ ] Implémenter la détection des collisions (`collision.js`)
- [ ] Système de score
- [ ] Effet de défilement / course
- [ ] Reset du jeu en cas de game over
- [ ] Afficher le joueur et le sol

---

## 🟧 Phase 4 – Contenu & Gameplay

**Objectif :** Ajouter les obstacles et les mécaniques

- [ ] Obstacles au sol (monuments de différentes tailles)
- [ ] Obstacles volants (ballon rugby, Beluga Airbus)
- [ ] Mécanique de glissade (slide avec skate)
- [ ] Double saut
- [ ] Salto acrobatique
- [ ] Vagues dynamiques (Quai de la Daurade)
- [ ] Étoiles et bonus (Cité de l’Espace)
- [ ] Configuration des niveaux (`levels.js`, `level1.js`, etc.)
- [ ] Drapeau de fin de niveau + animation victoire
- [ ] Système de bonus (invincibilité, multiplicateurs)

---

## 🟪 Phase 5 – Interface & Polish

**Objectif :** Finaliser l’expérience utilisateur

- [ ] Interface de démarrage complète
- [ ] Sélection du joueur (chocolatine classique, blanc, dorée)
- [ ] Sélection de la difficulté (Facile / Moyen / Difficile)
- [ ] Carte interactive pour choisir les niveaux
- [ ] Modes de jeu (Classique, Endless, Chrono, Bonus)
- [ ] Sauvegarde avec localStorage (choix, scores, niveaux débloqués)
- [ ] Animations fluides (jump, slide, salto)
- [ ] Intégration des sons et de la musique
- [ ] Responsive / adaptation écran
- [ ] Commenter le code

---

## 🟫 Phase 6 – Tests & Finalisation

**Objectif :** Un jeu stable et prêt à présenter

- [ ] Tester toutes les mécaniques
- [ ] Tester tous les niveaux
- [ ] Corriger les bugs
- [ ] Vérifier la lisibilité du code
- [ ] Vérifier les bonnes pratiques (nommage, fonctions)
- [ ] Optimiser les performances si besoin
- [ ] S’assurer qu’une version jouable et fonctionnelle existe
- [ ] Relire la documentation (.md)

---

## 🟥 Phase 7 – Présentation

**Objectif :** Livrer et présenter le projet

- [ ] Finaliser tous les fichiers .md de documentation
- [ ] Compléter « Qui a fait quoi » (projet-6.md)
- [ ] Préparer la présentation orale
- [ ] Expliquer les choix techniques et de design
- [ ] Expliquer le jeu et son fonctionnement
- [ ] Présenter la répartition des tâches
- [ ] **Présentation finale – 7 avril 2026** 🎯
- [ ] Rendre le projet (code + documentation)
